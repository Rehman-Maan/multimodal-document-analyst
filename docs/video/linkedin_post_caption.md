## Recommended Caption

I built **Multimodal Document Analyst**, a Django-based document AI workflow that turns PDFs and scanned files into structured, reviewed, exportable data.

This is not just a "chat with PDF" demo. I wanted to build something closer to a real document operations product:

- Upload PDFs and scanned documents
- Process files in the background with Celery
- Extract page text and document previews
- Run schema-based structured extraction
- Review uncertain fields before approval
- Ask grounded questions with focused citations
- Export reviewed data as JSON or CSV
- Evaluate extraction quality with precision, recall, F1, and exact match metrics

The stack includes Django, DRF, PostgreSQL, Redis, Celery, Docker Compose, OpenAI-ready services, pytest, Playwright, GitHub Actions, and Trivy.

This project helped me understand how document AI systems work beyond the model call: background jobs, data modeling, review workflows, citations, exports, evaluation, testing, and deployment readiness.

GitHub:
https://github.com/Rehman-Maan/multimodal-document-analyst
